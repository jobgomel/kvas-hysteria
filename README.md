# Kvas-Hysteria 🚀

Расширение для популярного менеджера обхода блокировок **Kvas** ([qzeleza/kvas](https://github.com/qzeleza/kvas)), добавляющее полную поддержку и автоматизацию протокола **Hysteria 2** на роутерах Keenetic с развернутой средой Entware.

Данный пакет позволяет использовать Hysteria 2 в качестве базового прокси-клиента (SOCKS5), бесшовно интегрируя его в экосистему Kvas и операционную систему KeeneticOS.

---

> **Внимание!** Проект не планировался для публикации и разрабатывался для личного пользования.

> **Примечание:** Тестировалось только на устройствах:  Ultra (NC-1812), Ultra (KN-1810), Skipper 4G (KN-2910)
> **Примечание:** Тестировалась только конфигурация с обфускацией **Salamander**

## 🛠 Установка
Запустите команду установки в терминале (SSH) вашего роутера. Скрипт создаст структуру папок, скачает необходимые компоненты и создаст глобальный симлинк `/opt/bin/kvas-hysteria`.

```
curl -sL https://raw.githubusercontent.com/jobgomel/kvas-hysteria/main/install.sh | sh
```

---

## 🚀 Использование и настройка
После успешного завершения установки управление пакетом осуществляется через единый CLI-интерфейс `kvas-hysteria`.

### 1. Импорт конфигурации (Добавление пира)
Скопируйте вашу ссылку `hysteria2://` из личного кабинета VPN или генератора конфигов и передайте её команде `add`.
```bash
kvas-hysteria add "hysteria2://user:password@your_server_ip:port/?sni=your_sni&obfs=salamander&obfs-password=secret"
```
⚠️ **Важно:** Всегда берите ссылку в двойные кавычки `""`, иначе спецсимволы (такие как `&` или `?`) будут неверно обработаны командной оболочкой роутера!

### 2. Управление службой
Вы можете вручную управлять состоянием демона Hysteria 2:
```bash
kvas-hysteria start    # Запустить прокси
kvas-hysteria stop     # Остановить прокси
kvas-hysteria restart  # Перезапустить процесс
```

### 3. Просмотр статуса
Чтобы проверить текущее состояние службы, версию бинарника, параметры активного сервера и пути конфигурации, выполните команду без аргументов:
```bash
kvas-hysteria
kvas-hysteria test     # Тестирование туннеля
```

---

## 🗑 Удаление
Если вам необходимо полностью удалить пакет, остановить службы и очистить конфигурацию роутера от созданного прокси-интерфейса, выполните:
```bash
kvas-hysteria uninstall
```

## ✨ Основные возможности

* **Полная автоматизация:** Скрипт самостоятельно определяет архитектуру процессора вашего роутера (ARM, MIPS, MIPSEL), находит и скачивает актуальный бинарный файл Hysteria 2 напрямую с официального GitHub.
* **Изолированная архитектура:** Пакет разворачивается в выделенную директорию `/opt/apps/kvas-hysteria/`, не засоряя системные папки Entware.
* **Умный импорт ссылок:** Парсинг конфигураций «на лету» из стандартных ссылок вида `hysteria2://` или `h2://`. Поддерживается обфускация **Salamander**.
* **Интеграция с KeeneticOS API (RCI):** Автоматическое создание, настройка и запуск независимого SOCKS5-интерфейса в панели управления Keenetic (раздел *«Другие подключения»*). Полная совместимость с KeeneticOS 3.x, 4.x и 5.x.
* **Безопасность памяти:** Встроенный модуль проверки дискового пространства защищает внутреннюю флеш-память роутера от переполнения и предупреждает пользователя о рисках.
* **Гибкие настройки туннеля:** Использование шаблона конфигурации QUIC (0-RTT Fast Open, Lazy mode, оптимизация таймаутов и контроль заторов BBR).

---

## 💾 Требования к памяти

Бинарный файл Hysteria 2 весит порядка **22 MB**. Установка пакета во внутреннюю память роутера (UBIFS) может привести к нестабильности KeeneticOS в случае её нехватки.

В скрипт встроена автоматическая проверка:
* **На USB-накопителе:** Требуется минимум **25 MB** свободного места.
* **Во внутренней памяти:** * Если свободно **менее 30 MB** — установка жестко блокируется во избежание сбоев KeeneticOS.
    * Если свободно **от 30 до 40 MB** — скрипт выдаст предупреждение и потребует ручного подтверждения `[y/N]`.
    * *Рекомендуется использовать Entware, развернутый на USB-флешке.*
